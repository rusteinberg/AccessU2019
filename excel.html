Accessibility Considerations

  • Keyboard Only
  • Screen Magnification
  • Use of Color
  • Screen Reader Friendly
  • Cognitive Load

Why we're doing what we're doing and who it affects.

  • Keyboard only users -- Excel keyboard accessible if it's in the grid.
  • Screen magnifier users -- People with low vision or blocked vision.
  • Use of color -- Be aware not everyone sees color the same way.
  • Screen Reader Friendly -- What is a screen reader? Will spend time making Excel files readable for screen readers.
  • Cognitive Load -- Simplifying tables and presentation will make the info easier to quickly consume for everyone.

General Tips

  • Make sure your language is set in Excel properties/complete Metadata
  • Make sure sheets have unique names and delete unused sheets
  • If a sheet has multiple tables, blank row between each. Best practice one table per sheet.
  • Complex tables can only be made accessible if Excel version converted to PDF or HTML
  • Remove comments
  • Don't embed multimedia
  • If your sheet needs instructions, put them in A1 cell
  • Set zoom level and focus on starting cell for each sheet (usually A1)
  • Add alternative text for charts and graphs--more on this later

Spacing and Fonts

Optimize for users who use screen magnification.

Another consideration is to optimize for screen magnification. Many people with low vision will use a screen magnifier to read documents. And any anyone accessing a document on a phone or tablet will have a limited viewport, as well.

When using a limited view port, a user can only see a few cells at a time, and will start at the top and pan left to right.

For example, when this table with a limited viewport, right off, they would see an empty column A, read the titles rows 1 and 2, and then empty rows 3 and 4. That's a lot of blank space to wade through to get to the actual information on this sheet. So it's best to eliminate blank rows and columns in excel. And It's also important to check is that the information makes sense when read left to right top to bottom.

Another thing that happens under screen magnification is that fonts begin to pixelate. Therefore it's important to use a sans serif font like Arial or Verdana. Also the font need to be a minimum of size 12.

Lastly, you need to use strong contrast to maximize readability. The headers on this table do not have enough contrast between the background color and the text color.

Overall, the original version of this table is pretty difficult for anyone to read. The same table was then optimized for accessibility.

  • Empty rows and columns were deleted.
  • The font was changed to Arial and bumped up to size 12.
  • The color contrast was fixed on the headers.

You will probably agree that these changes have made the table much more readable for anyone, not just thse with visual impairments.

How to Set the Default Font

Save time by making the fonts in your Microsoft Excel program accessible by default.

Just go to File > Options > General > and look for "When Creating New Workbooks..."

Choose a sans serif font such as Verdana or Arial and choose font size 12.

Also be sure to always set the zoom to 100% or the font won't really be the size you've set them at. And set first cell at A1 on first worksheet.

Simplyfying Tables

  • Use a simple table structure and specify row and column header information.
  • If your table is not structured like that, you need to rethink it.

If a table is nested in another table or if a cell is merged or split, a screen reader loses count of table cells and can't provide helpful information about the table after that point.

Importance of Table Headers

  • When creating a table from scratch, Excel allows you to define table headers.
  • This information is not communicated to all screen reader users.

Tables are difficult for screen reader users to navigate whether they appear on a web page, a Word document, a PDF, a PowerPoint, and yes in Excel.

The main issue is screen reader users need to be able to associate column and row headers with the cell they are in.

It's Ok to have merged cells above data. But if a table has table headers with merged cells, it is difficult if not impossible for screen reader users to know where they are.

Although Excel has a way of defining a top row as a header row, that information is not communicated to all screen reader users. Demo on JAWS.

Using Name Manager

  • Microsoft Excel has a built-in function called Name Manager that can be used to give names to a cell or range of cells.
  • Using a special naming convention, JAWS and NVDA can recognize table header rows and columns. Microsoft reports that Narrator also recognizes this coding.
  • Determine if table has column headers, row headers, or both column and row headers
  • Define regions as long as they begin with the string Title, ColumnTitle, or RowTitle.
  • Each name in a workbook no matter what sheet it is on must have a unique name.

Column Title Steps

In practice file, this is on Sheet 1, 1. Contact List

  1. Examine table to determine starting cell of column headers
  2. Select starting cell (in example it is A2)
  3. Go to Formulas tab, then Name Manager, then New (keyboard shortcut Control+F3)
  4. In Name, type ColumnTitle (use camel case)
  5. If your workbook has more than one table like this, one table would use ColumnTitle1 and the other ColumnTitle2
  6. Optional: Leave a comment in Comments to not delete
  7. In Scope, select Workbook
  8. Ensure Refers to says, ='1. Contact List'!$A$2
  9. Select OK button, then Close button for Name Manager

  • You may use this technique throughout a workbook containing many worksheets, as long as each name is unique.

Intro to Color Blindness

When color is included in your spreadsheets, you need to make sure that information is not conveyed with color alone. This is because not everyone sees colors the same way, and also not everyone can see. Those who can't see, use screen readers that read the information on the screen aloud. Let's look at an example.

Here is a table with a date column that has past due thresholds, indicated by blue and green backgrounds. Presumably there would be a legend that would explain exactly what the blue and green shading mean. The problem is that some users may not be able to distinguish between the blue and green colors; and people who use screen readers would hear the text in the cells but would not be aware of the background colors at all. So this is an example of how using color to convey information can be inaccessible.

But... with some small tweaks, this table can be made accessible. For example, you could add a column that *says* what is past threshold – in this table it is either a diagnostic, an assessment, or neither. For those who can distinguish the color shading, it is much more obvious now that blue means (diagnostic) and green means (assessment). And for those who can't rely on color, the information is clearly communicated in text. This table is now accessible.

By the way, watch out for color in charts too. For example, in a pie chart, you can add labels and values next to each wedge so that users who are color blind can have an alternate way to get the information.

The default settings of the Chart Wizard in Excel are not color accessible. Use the formatting tools to change line styles and colors.

Pie Chart Examples

Keep in mind that some of your Excel-generated charts and graphs may also be used for presentations in small to large groups. Having said that, make sure that associate a number with each bar in your graph along with the name.

Microsoft guide to writing alt text for Excel

  • All charts need something called Alt Text. This is a text description of what the chart is about.
  • Type 1-2 sentences in the text box to describe the object and its context to someone who cannot see it.
  • Do not start your alt text with, "This is an image of ..." Screen readers will identify images so it would be redundant to hear "Graphic: This is an image of..."
  • Microsoft has recently updated Office 2016 to make writing alt text easier. Right click on the chart and select "Edit Alt Text"
  • If your organization has not updated Office 2016 yet, right click on the chart, select Format Chart Area or Format Picture, select the Size and Properties icon (far right), then select Alt Text. Leave Title blank
  • Office 365 has an auto alt text generator that uses artificial intelligence to describe your picture. To turn it off: Select File > Options > Ease of Access. In the Automatic Alt Text section, select or unselect the Automatically generate alt text for me option, and then select OK.
  • In addition, if your artwork is decorative, in Office 365 and Office 2019 you can mark it as decorative.
  • If it is decorative in Office 2016, type a pair of quotation marks ("") or a space in the alt text field
